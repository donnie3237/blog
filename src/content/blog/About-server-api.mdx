# About server api

ฉันคิดว่าการทำ เซิฟเวอร์นั้นฟังดูน่าสนุก แต่ฉันพบอะไรบางอย่างหลังจากที่เริ่มเขียนโค้ดฝั่งเซิฟเวอร์ไป และนี่คือความเห็นของฉัน

## Clean architecture

ผู้คนมีแนวคิดที่แตกต่างเกี่ยวกับเรื่องนี่ แต่การเขียนโค้ดให้ มี Linter มีแบบ MVC MVVM DDD ที่ทำให้กลับมาอ่านได้ง่าย และทำให้ดูแลต่อได้ง่ายนั้นค่อนข้างจำเป็น

## Algorithm

อัลกอริทึมที่ดี จะทำใหโค้ดที่เขียน มีประสิทธิภาพเพิ่มขึ้น แถมยังลดความเสี่ยงจากการมีช่องโหว่ การให้ความสำคัญเรื่อง BigO เพื่อลดการทำงานของโปรแกรม หรือแม้แต่ การเปลี่ยน framework ที่มีประสิทธิภาพมากขึ้น การเข้าใจการดีไซน์ middleware ก็เป็นสิ่งสำคุญที่จะคอยตรวจจับปละ ดพิ่มความปลอดภัยของเซิฟเสอร์ได้

## SwaggerUI

มันค่อนข้างลำบากถ้าไม่เขียนอะไรซักอยางไว้เกี่ยวกับ api และตัว  SwaggerUI ได้เข้ามาตอบโจทย์ สำหรับเช็คว่ามีพาทไหนบ้าง มี key ตัวไหนที่ต้องใช้กับอะไรบ้าง มีโมเดลข้อมูลเป็นแบบไหน ฉันคิดว่า swaggerUI ค่อนข้างตอบโจทย์จริงๆ

## Tracker

ฉันได้ลองใช้ sentry เพื่อตรวจจับบัคที่เกิดขึ้นและมันจะทำการแจ้งเตือนไปยังอีเมลของฉัน มันค่อนข้างเจ๋งเลย เมื่อ สิ้นเดือน มันจะสรุปข้อมูลว่า มีพาธไหนถูกเรียกใช้ไปกี่ครั้ง และใน 1 เดือนนั้นมีอะไรเกิดขึ้นบ้าง
