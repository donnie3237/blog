# UI framework

การตัดสินใจในการสร้างเว็บหรือแอพลิเคชั่นของเราในตอนนี้มันค่อนข้างมีหลายวิธีจนเกินไป และมันค่อนข้างลำบากในการตัดสินใจที่จะเลือก และนี่อาจเป็นตัวอย่าง

## Web Framework

* React ผมค่อนข้างมั่นใจว่า ตัวนี้ค่อนข้างดี และน่าจะใช้งานได้อีกยาวนาน เพราะว่ามีคอมมูนิตี้เยอะมาก และมันเป็นตัวที่ผมใช้ค่อนข้างบ่อย มันถูกพัฒนาหลักโดย Meta แต่สามารถมีปลั้กอินหลายๆตัวที่เข้ากันได้กับตัวนี้
* Solid ตัวนี้พัฒนาต่อมาจาก React  เพิ่มเติมเรื่องประสิทธิภาพด้วยการใช้ signal ทำให้ไม่ต้องรีเรนเดอร์หลายรอบที่จะส่งผลเสียต่อประสิทธิภาพของการ
* Svelte ตัวนี้ที่ออกมาตอนแรกดูเหมือนจะมีประสิทธิภาพดี และเขียนเข้าใจง่ายมาก แต่หลังๆ เริ่มเพิ่มฟีเจอร์ ทำให้ดูเริ่มยากในการเขียนขึ้น แต่ผมไม่ค่อยได้เเตะตัวนี้มากเท่าไหร่ จึงพูดไม่ได้เยอะ
* Vue ตัวนี้มีคอมมูนิตี้ที่ใหญ่ แต่ผมมีประสบการณ์กับเฟรมเวิร์คตัวนี้น้อยมาก เลยอาจพูดได้ไม่ค่อยเยอะ แต่ว่าเท่าที่ฟังมา เขาบอกว่า เขียนค่อนข้างง่าย เข้าใจได้ง่าย
* Nextjs ในฝั่งของ rendering framework ในตัวของ nextJS ผมคิดว่า พัฒนามาได้ดีมาก ใช้งานง่าย คอมมูนิตี้เยอะ ทุกอย่างมันค่อนข้างโอเคเลยสำหรับการใช้งานจริง
* Astro ตัวนี้เพิ่งมาใหม่ แต่ค่อนข้างใช้งานง่ายเหมือนกันแต่ ไม่ค่อยเห็นใครใช้ใน โปรเจคงานจริง ส่วนใหญ่จะเปผ้นพวก เว็บ บลอคส่วนตัว หรือ พอร์ตฟอลิโอมากกว่า

## CSS framework

* Tailwindcss เฟรมเวิร์คนี้ ผมใช้งานบ่อยและยังคิดว่ามันยาวไป ส่วนตัวคิดว่า การใช้พวก DaisyUI หรือ DoseUI เข้ามาใช้งานร่วมด้วย อาจเป็นทางเลือกที่ค่อนข้างเยี่ยมเลย
* Bootstrap เฟรมเวิร์คนี้ค่อนข้างอยู่มาเก่าแก่ มีความน่าเชื่อถือสูงเลยทีเดียวเชียว 
* MUI ตัวนี้ใช้กับ React ถือเป็นแนว Style components ที่ค่อนข้างดีเลย มีหลากหลายตัวให้เลือกใช้
* StyleX เฟรมเวิร์คตัวนี้พัฒนาโดย meta ที่มีความเชื่อว่าการเขียนแบบ style component เป็นอะไรที่ดี และmeta ก็ใช้เองด้วย
